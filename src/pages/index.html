¿Como probar la API?

Para probar la API proveemos los siguientes endpoints:

Ususarios:

  • GET /api/users
  • GET /api/users/:id
  • POST /api/users
    Ejemplo de body 
                    {
                  "name": "Prueba",
                  "email": "prueba@example.com",
                  "password": "secreto456",
                  "role": "user",
                  "active": true
                }
  • PUT /api/users/:id
  • DELETE /api/users/:id

Vendors:

  • GET /api/vendors
  • GET /api/vendors/:id
  • POST /api/vendors
    Ejemplo de body 
                    {
                  "name": "CyberGuard Inc.",
                  "country": "USA",
                  "supportEmail": "support@cyberguard.example",
                  "phone": "(305) 555-1234",
                  "supportHours": "Mon–Fri 9-18",
                  "rating": 4.2,
                  "active": true
                }
  • PUT /api/users/:id
  • DELETE /api/users/:id

Planes

  • GET /api/plans
  • GET /api/plans/:id
  • POST /api/plans
    Ejemplo de body 
                    {
                  "name": "Mobile Lite",
                  "vendorName": "CyberGuard Inc.",
                  "price": 9.99,
                  "os": ["android", "ios"],
                  "rating": 3.9,
                  "features": ["antivirus", "antiphishing"],
                  "description": "Protección esencial para móviles."
                }
  • PUT /api/users/:id
  • DELETE /api/users/:id
